// 2.1 РИБ (Распределенная информационная база)

// Обмен только между полностью идентичными конфигурациями.
// Обновления можно вносить только в центральную базу (корневой узел).
// Дочерние базы (дочерний узел) обнавляются при получении сообщения от центральной базы.
// Пример: Магазины, каждый магазин работает в отдельной ИБ, но конфигурации все идентичные.

//============================================================================

// Создание плана обмена и дочерних баз:
// 1. Создаем план обмена "ОбменСМагазинами".
// 2. Вклюючаем настройку "Распределенная информационная база".
// 3. Включаем в состав нужные документы, кликаем ПКМ => Подобрать связанные для всех.
// 4. Запускаем 1С:Предприятие => план обмена "ОбменСМагазинами" => Создаем узлы:
// - Корневой узел => Код "ЦБ", Наименование "Центральный офис".
// - Дочерний узел => Код "М1", Наименование "Магазин №1".
// - Дочерний узел => Код "М2", Наименование "Магазин №2".
// 5. Создаем информационные базы дочерних узлов командой "Создать начальный образ".
// 6. Добавляем информационные базы дочерних узлов в список баз 1С:Предприятия.

// Отправка данных из Центральной базы для Дочерней Базы:
// 1. Запускаем 1С:Предприятие Корневого узла, заходим в план обмена.
// 2. Выбираем дочерний узел "Магазин №1" и жмем на команду "Записать изменения".
// 3. Выбираем каталог, куда будет сохранет zip-файл с изменениями для дочерней базы.

// Сохранение данных в Дочерней базе, отправленных из Центральной базы:
// 1. Запускаем 1С:Предприятие Корневого узла, заходим в план обмена.
// 2. Выбираем корневой узел "Центральный офис" и жмем на команду "Прочитать изменения".
// 3. Выбираем zip-файл, предназначенный для этой дочерней базы.

// Отправка сообщения(квитанции) для Центральной базы, для очистки таблицы изменений:
// 1. Запускаем 1С:Предприятие Корневого узла, заходим в план обмена.
// 2. Выбираем корневой узел "Центральный офис" и жмем на команду "Записать изменения".
// 3. Выбираем каталог, куда будет сохранет zip-файл с квитанцией для центрального офиса.
// 4. Запускаем 1С:Предприятие Корневого узла, заходим в план обмена.
// 5. Выбираем дочерний узел "Магазин №1" и жмем на команду "Прочитать изменения".
// 6. Выбираем zip-файл с квитанцией, предназначенный для центрального офиса.
// 7. После чтения квитанции, ненужные записи таблицы изменений будут удалены.
