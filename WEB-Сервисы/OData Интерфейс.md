```bsl
// Интерфейс OData - Возможность получение данных Базы 1С через протокол HTTP


// Правила формирования имени ресурса:
// Справочник				- Catalog
// Документ					- Document
// Журнал документов		- DocumentJournal
// Константа				- Constant
// План обмена				- ExchangePlan
// План счетов				- ChartOfAccounts
// План видов расчета 		- ChartOfCalculationTypes
// План видов характеристик	- ChartOfCharacteristicTypes
// Регистр сведений			- InformationRegister
// Регистр накопления		- AccumulationRegister
// Регистр расчета			- CalculationRegister
// Регистр бухгалтерии		- AccountingRegister
// Бизнес-процесс			- BusinessProcess
// Задача					- Task


// Параметры:
// $format=atom - возвращает данные в формате atom-xml
// $format=json - возвращает данные в формате json
// $metadata    - указывает на требование описания интерфейса Odta
// $select		- описывает список свойств сущности, для получения
// $filter		- описывает отбор, при получении данных


// Правила формирования условия отбора:
// =================================================================================
// | Описание		  | Имя | Пример											   |
// |==================|=====|======================================================|
// | Равно			  | eq 	| /Catalog_Города?$filter=Description eq 'Главный'	   |
// | Не равно		  | ne 	| /Catalog_Города?$filter=Description ne 'Пермь'	   |
// | Больше           | gt 	| /Catalog_Товары?$filter=Цена gt 100				   |
// | Больше или равно | ge 	| /Catalog_Товары?$filter=Цена ge 100				   |
// | Меньше			  | lt 	| /Catalog_Товары?$filter=Цена lt 100				   |
// | Меньше или равно | le 	| /Catalog_Товары?$filter=Цена le 100				   |
// | Логическое Или	  | or 	| /Catalog_Товары?$filter=Цена lt 10 or Цена gt 1000   |
// | Логическое И	  | and	| /Catalog_Товары?$filter=Цена gt 10 and Цена lt 1000  |
// | Отрицание		  | not	| /Catalog_Товары?$filter=not (Цена eq 1000)		   |
// =================================================================================


// 1. Конфигуратор => Администрирование => Публикация на веб сервере => Odata => Вкл.
// 2. Запускаем 1С:Предприятие, открываем обработку "СоставОбъектовДляИнтерфейсаOData"
// 3. Определяем состав объектов, которые будут доступны в запросах к базе
// 4. Метод для состава: УстановитьСоставСтандартногоИнтерфейсаOData(МассивОбъектов)
// 5. Если выполняем запросы через Postman - потребуется Basic Auth (авторизация)


// Получение списка контрагентов в формате JSON (Метод GET):
// http://localhost/Education/odata/standard.odata/Catalog_Контрагенты?$format=json

// Получение контрагента по ИНН в формате JSON (Метод GET):
// /Catalog_Контрагенты?$format=json&$filter=ИНН eq '165161646'

// Получение контрагента по GUID в формате JSON (Метод GET):
// /Catalog_Контрагенты(guid'adda5c9a-5b1e-11ef-a453-2cf05d8110c7'?$format=json

// Создание контрагента (Метод POST):
// http://localhost/Education/odata/standard.odata/Catalog_Контрагенты?$format=json
// Тело запроса (Body):
{
	"Description": "Новый контрагент Odata",
	"ВидКонтрагента": "ФизическоеЛицо",
	"ИНН": "165161646",
	"Покупатель": true,
}

// Изменение контрагента (Метод PATCH):
// /Catalog_Контрагенты(guid'adda5c9a-5b1e-11ef-a453-2cf05d8110c7'?$format=json
// Тело запроса (Body):
{
	"Description": "Обновленный контрагент Odata",
	"ВидКонтрагента": "ФизическоеЛицо",
	"ИНН": "165161646",
	"Поставщик": true,
}

// Удаление контрагента (Метод DELETE):
// /Catalog_Контрагенты(guid'adda5c9a-5b1e-11ef-a453-2cf05d8110c7'?$format=json
```
